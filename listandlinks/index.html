Creating a Webpage on the Production Server

Step by Step

  1. After cloning your repository and accessing it through terminal, create two new gedit files named "index.html" and "main.css"
  2. Edit your webpage using HTML code for index, then link it with main.css and use CSS to customize the look of your website.
  3. Save your files, and use the git commands to save it on GitHub for safety.
  4. In terminal, access the production server using "sftp [name-id]@cmis-hsteacher". Add the production server to your known servers if it's the first time accessing it, and enter your password. Be sure to have moved to the location of your file before accessing the servers
  5. Once in the production server, move into the "public_html" folder.
  6. You can add files using "put [file name]" and remove them using "rm [file name]".
  7. Whilst having your terminal out with the production server accessed, you can view your work by opening a browser and typing "cmis-hsteacher/~[name-id]" in the address bar.

Creating List and Links

  • Create a new folder called "listsandlinks" in your repository folder.
  • Inside listsandlinks create a file called "index.html" and a file called "main.css"
  • In index.html create an ordered list of step by step instructions to create a new webpage on the production server.
  • Also create an unordered list of 5 or more links to websites with useful webdesign related information.
  • In the nav section, create an unordered list of links with a link back to your main index page.
  • In your main index.html file in public_html, add an unordered list of links to the nav section with a link to "listsandlinks".
  • In your old index.html, add an article with a heading and a paragraph description of this assignment. Also add an unordered list that contains this list of requirements.
  • Turn in your work.
